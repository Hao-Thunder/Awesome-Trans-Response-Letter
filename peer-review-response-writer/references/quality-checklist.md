# Quality-Control Checklist

Use this checklist before finalizing a response or full response letter.

## Scientific Accuracy

- [ ] The response answers the actual concern rather than a nearby issue.
- [ ] Every technical claim is supported by supplied or verified evidence.
- [ ] Planned work is not described as completed.
- [ ] Numerical values, citations, equations, and implementation details are not invented.
- [ ] Empirical evidence is not described as mathematical proof.
- [ ] Claims are limited to the tested or derived scope.

## Response Logic

- [ ] The direct answer appears near the beginning.
- [ ] The selected stance is clear: agree, partially agree, disagree, already addressed, or evidence required.
- [ ] The explanation connects the reviewer concern to the evidence.
- [ ] The manuscript revision directly resolves the concern.
- [ ] Compound comments are addressed separately when needed.

## Revision Traceability

- [ ] Completed changes are described factually.
- [ ] Proposed changes are labeled as proposed.
- [ ] Revised text matches the response claim.
- [ ] Section, page, line, figure, and table locations are verified or left as placeholders.
- [ ] All placeholders are visible and searchable.

## Tone and Style

- [ ] The tone is respectful and non-defensive.
- [ ] The response does not blame the reviewer for misunderstanding.
- [ ] Gratitude phrases are not repeated mechanically.
- [ ] The language is concise and specific.
- [ ] Translation preserves uncertainty and technical meaning.

## Full-Letter Consistency

- [ ] Reviewer and comment numbering are preserved.
- [ ] Terminology is consistent across all responses.
- [ ] Experiment settings and reported values do not conflict.
- [ ] Revisions promised to different reviewers are compatible.
- [ ] Abstract, contribution, limitation, and conclusion claims remain aligned.
- [ ] Remaining actions and unresolved placeholders are consolidated.
